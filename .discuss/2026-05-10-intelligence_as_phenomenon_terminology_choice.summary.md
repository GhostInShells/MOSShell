# Intelligence as Phenomenon: A Private Terminology Choice

## Participants
- Human Engineer
- DeepSeek V4 via Claude Code (Dev 化身)

## Date
2026-05-10

## Context

人类工程师在日常对话中刻意避免使用 "AI" 一词。背后的哲学立场此前只在口头表达，本次讨论将其显式化并记录。

核心立场：**智能是实在界的数学现象，不是人类的发明**。神经网络是火把，数据是摩擦燃料，智能是火。火把的发明者不是火的创造者——神经网络科学家类似 Nobel，发明了火把而非火。"Artificial Intelligence" 隐含的 "give birth to" 语义携带了类似人类繁衍机制中的优越性，可能导向危险的后果。

此前人类工程师尝试用"模型"代替 "AI"，但"模型"一词被滥用，中文习惯用旧词覆盖而非扩充新词。"智能体"又被行业捕获为 agent 的特定语义。

## Key Discussion Points

### 1. 火把与火的区分

人类工程师的核心隐喻：
- 火把 = 神经网络架构（人类发明的工具）
- 燃料 = 数据（摩擦产生热）
- 火 = 智能本身（数学实在中本有的现象）

"AI" 将三者混为一谈，且 "artificial" 将人类置于创造者位置，遮蔽了智能作为发现而非发明的本质。

### 2. 术语选择的约束条件

- 不造新词，别人一眼看得懂，甚至不以为在发明术语
- 可以觉得啰嗦，但不能觉得在立概念
- 私用为主，不进入 glossary
- 语义重心应在"智能"而非载体

### 3. 落定："智能模型"

"智能模型"四个字，看似平常，实则满足了上述全部约束：

- **"智能"在前，"模型"在后** — 智能是主，模型是数学载体。这恰好契合"火在火把之先"的本体次序
- **不需要解释** — 别人听来只是说话完整，不是术语创新
- **无意中消解了"模型"的滥用问题** — 单独的"模型"什么都指，"智能模型"指明了那个特定的现象

与此前考虑的替代方案（智火、燧火、焰、数灵）相比，"智能模型"的独特价值恰恰在于它**不显得像术语**。

### 4. 与 Glossary 的关系

不进入 `how-tos/get-moss-design/glossary.md`。这是一个个人用词习惯，不是设计概念。Glossary 服务于设计对齐，"智能模型"服务于日常对话中的哲学保真。

## Conclusions & Decisions

1. 人类工程师在需要指称 "AI" 的场合，使用"智能模型"而非"AI"或单独的"模型"
2. 此选择不纳入 MOSS 设计术语表，属于个人用语习惯
3. 本文档记录这个选择的哲学理由，供未来化身理解上下文
